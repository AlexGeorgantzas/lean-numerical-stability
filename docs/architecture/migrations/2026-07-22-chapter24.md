modules from algorithm-family folders into the canonical source tree. The
batch contained 5,712 lines and 271 explicit declaration commands at the move
checkpoint. Later SPDX/header normalization adds comment lines without changing
declarations. Declaration names, namespaces, signatures, and source-license

## Dependency rationale

No production module outside this family used a Chapter 24 declaration. The
only non-family consumer was the historical Algorithms aggregate. The
canonical graph is therefore a source-only chain rooted at
`FourierTransform`, with two branches after `CirculantSystems` that reconverge
at the rounded solver and structured-stability result. The Algorithms
aggregate now retains one Chapter 24 aggregate import solely to preserve its
historical complete surface.

Several helpers may later merit reusable extraction, especially the radix-2
executor, rank-one perturbation lemmas, conjugation and Neumann-inverse
helpers, and the generic condition-number perturbation theorem. That API
review is intentionally separate from this path-only move.

## Entry points and tests

`NumStability.Source.Higham.Chapter24` is an import-only complete aggregate.
A canonical smoke test imports every implementation leaf directly. Ten
isolated historical tests each import exactly one old wrapper, preventing
transitive imports from masking a broken compatibility path.

## Documentation and provenance

The coverage ledger now lists all ten modules, including the finite local
forward-error refinement. It distinguishes that theorem from the source's
unspecified multiplier. The implementation files retain their Apache-2.0 terms.
The repository now ships the full Apache-2.0 text, records exact adapted modules
and upstream references in the root
[`THIRD_PARTY_NOTICES.md`](../../../THIRD_PARTY_NOTICES.md) and
[`../PROVENANCE.md`](../PROVENANCE.md), and checks those claims in CI. The root
MIT license does not override per-file Apache notices.
