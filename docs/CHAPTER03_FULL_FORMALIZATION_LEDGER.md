## Hidden-Hypothesis Audit

- Several Lean theorems use non-strict `<=` local-error bounds, matching the
  repository `FPModel`; source statements often write strict `<`.
- Problem 3.3 explicitly exposes the denominator-separation hypothesis
  `muNext < |qhatNext|`, which is necessary for the inherited-error denominator
  to be positive.
- Problem 3.6's lower bound is proved for the relative majorant `G=|A||B|`.
  The source exercise writes general nonnegative `E,F`; Lean does not overclaim
  that arbitrary `E,F` give the same residual majorant without the corresponding
  feasibility hypotheses.
- Problem 3.7 is the algebraic complex transfer layer. It assumes a complex
  dot-product relative-error expansion with a supplied radius; the scalar
  complex operation constants are already formalized in `ComplexArithmetic.lean`.
- Problem 3.8 avoids informal asymptotic notation by proving a finite-budget
  theorem. The source `k n^2 u + O(u^2)` is obtained by instantiating the local
  budget `beta` with the matrix-multiply first-order budget.
- Problem 3.9 is closed at the standard-model error-bound level. The guard
  digit note is not formalized as a separate concrete finite-format theorem.
- Problem 3.11 remains the only full-PDF theorem-bearing gap: exact arithmetic,
  the finite trace surface, the initial IEEE-double square, the exact first
  square root for the six source inputs, terminal IEEE-double fixed points, the
  first thirty predecessor-square transitions through the exact
  non-endpoint state `1 - 1073741768 * 2^-53`, the one-step square
  underflow-to-zero terminal theorem, the represented four-decimal display rows,
  and two conditional `m=75` phase-law explanations are closed, but the
  platform-specific remaining rounded traces are not.  The
  open theorem family is tracked in
  `docs/CHAPTER03_PROBLEM3_11_BOTTLENECK.md`.
