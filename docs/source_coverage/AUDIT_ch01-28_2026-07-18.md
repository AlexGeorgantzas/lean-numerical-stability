## Closure standard

The audit uses the project skill's default core mode. A chapter is `PASS` only
when every selected named theorem, lemma, corollary, and algorithm, together
with every selected numbered equation and precise central prose claim, has a
compiled theorem at the printed strength. In particular:

- an honest `PARTIAL`, conditional transfer, or exact-arithmetic subcase is
  valuable formalization, but it does not close a stronger selected row;
- a hypothesis that packages the desired error budget, representability fact,
  or other mathematical crux does not count as a derivation of that crux;
- implementation-facing claims require a bridge from the actual rounded
  executor, not only an abstract envelope or exact transform;
- `DEFER-MISSING-PRECISE-STATEMENT` is accepted only where the source itself
  does not print a stable proposition to formalize.

These rules are stricter than the previous README convention, which counted an
"honest, documented partial" as a passing selected-scope row. That convention
is not the skill's closure rule and is retired by this audit.

## Method

1. Fetched and fast-forwarded local `main` to the remote baseline above.
2. Rebuilt the chapter inventory from the source and the skill rather than
   trusting existing PASS labels.
3. Compared source statements with actual Lean declaration types, including
   hidden hypotheses and executor-to-contract handoffs.
4. Rendered and visually checked source pages for disputed statements,
   including Theorems 2.4, 7.8, 9.3, 11.8, 20.7, equation (13.23), and the
   Chapter 27 two-pass norm algorithm.
5. Audited top-level imports and claimed cross-chapter bridges separately from
   theorem strength.
6. Scanned tracked Lean sources for proof escapes and sampled headline
   declarations with `#print axioms`.
7. Implemented tractable missing semantic bridges, then reran focused and full
   verification as recorded below.
