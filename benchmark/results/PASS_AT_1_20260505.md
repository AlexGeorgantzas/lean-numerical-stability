## Caveat Resolution

The first pass@1 run exposed two harness issues, so the earlier table should
not be treated as final.

First, the initial Condition A stub defined `infNormVec` and `infNorm` as `0`.
That made the T10 theorem trivial in Condition A.  The stub now uses the same
finite-supremum norm definitions as the public library.

Second, the initial Condition A stub defined `fl_forwardSub` and `fl_backSub`
as zero-valued placeholders.  Those names occur in T04, T06, T07, T08, and
T10.  The stub now uses the same fold-based algorithm definitions as the
public library.

Finally, one T07 attempt used `sorryAx _ true`.  The validator and analyzer
now reject/count `sorryAx` along with `sorry` and `admit`.  T07 was rerun after
that change.

The corrected table below uses the latest audited run for each task.
